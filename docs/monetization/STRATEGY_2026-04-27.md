❌ Paywall mostrado **sin contexto de valor**  
❌ Usuario **no sabe cuánto ahorró**  
❌ No hay **trigger claro** para mostrar upgrade

### Solución
✅ Mostrar **contador de ahorro acumulado**  
✅ Paywall **después de 2-3 búsquedas exitosas**  
✅ Mensaje **contextual** ("Ya ahorraste $180")

```
Usuario busca "Leche"
```

## 🔧 IMPLEMENTACIÓN TÉCNICA (ORDEN)

### Fase 1: Backend & Database (8h)
```
1. Crear tabla `user_savings` con RLS
   - user_id, product, savings_amount, search_at
   
2. Endpoint POST /api/v1/savings
   - Registra ahorro de cada búsqueda
   - Calcula total mensual
   
3. Endpoint GET /api/v1/savings/summary
   - Devuelve: total_acumulado, último_mes, trend
   
4. Endpoint POST /api/v1/premium/webhook (PayPal)
   - Actualiza profiles.plan = 'premium'
   - Actualiza profiles.premium_until
   
5. Middleware: check isPremium(user)
   - Valida acceso a features premium
```

### Fase 2: Frontend UI (6h)
```
1. Component SavingsCounter
   - Muestra ahorro acumulado en tiempo real
   - Trend visual (gráfica mini)
   
2. Component PaywallContextual
   - Aparece en ResultsScreen
   - Trigger basado en reglas
   
3. Screen PremiumStatus
   - Muestra estado actual (free/premium)
   - Fecha de expiración si premium
   - Botón de cancelación
```

### Fase 3: PayPal Integration (4h)
```
1. Crear planes en PayPal Live
   - ID: PREMIUM_MONTHLY
   - ID: PREMIUM_YEARLY
   
2. Configurar webhooks en PayPal
   - BILLING.SUBSCRIPTION.CREATED
   - BILLING.SUBSCRIPTION.UPDATED
   - BILLING.SUBSCRIPTION.CANCELLED
   
3. Mapear webhook → Supabase
   - Crear subscription record
   - Actualizar profiles.plan
```

### Fase 4: Testing & Analytics (4h)
```
1. A/B testing setup
   - Variante A: USD $1.99
   - Variante B: USD $3.99
   - Track: conversion_rate, churn, mrr
   
2. Analytics events
   - event: paywall_shown (reason)
   - event: paywall_clicked (accepted/rejected)
   - event: payment_started
   - event: payment_completed
   - event: payment_failed
```

## ⚠️ ANTI-PATTERN (QUÉ NO HACER)

❌ **Bloquear búsqueda sin Premium**  
→ Mata engagement, usuarios van a competencia

❌ **Mostrar paywall en primera búsqueda**  
→ Conversion < 0.5%, frustración pura

❌ **Ocultar alertas en Free (feature crítico)**  
→ Usuarios no ven valor antes de pagar

❌ **Pricing agresivo ($9.99/mes)**  
→ No hay ROI vs ahorro real ($300-800/mes es mucho)

❌ **No mostrar ahorro acumulado**  
→ Usuario no sabe por qué debería pagar

✅ **Usar ahorro REAL como el mejor argumento de venta**
